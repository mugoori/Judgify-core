# Security Policy

## 🔐 보안 취약점 보고

Judgify-core 프로젝트의 보안 취약점을 발견하셨다면, 공개 이슈로 보고하지 말고 다음 방법으로 비공개 보고해주세요.

### 보고 방법
- **이메일**: mugoori@github.com (보안 담당자)
- **제목**: `[SECURITY] 취약점 설명`

### 포함할 정보
- 취약점 상세 설명
- 재현 단계
- 영향받는 서비스/버전
- 가능한 경우 PoC (Proof of Concept)

### 대응 절차
1. **24시간 내** 수신 확인
2. **72시간 내** 초기 분석 및 심각도 평가
3. **7일 내** 패치 개발 및 테스트
4. **14일 내** 보안 업데이트 배포

## 🛡 보안 기능
- AST 기반 Rule Engine (코드 인젝션 방지)
- JWT 인증
- RBAC 권한 관리
- Pydantic 입력 검증
- SQL Injection 방지

|------|-----------|
| 2.0.x | ✅ 지원 |
| < 2.0 | ❌ 미지원 |
