## 1. 시작하기

### 1.1 시스템 요구사항

**최소 사양**:
- **OS**: Windows 10 (64-bit) 이상
- **RAM**: 4GB
- **디스크 공간**: 500MB
- **프로세서**: Intel Core i3 또는 동급
- **인터넷**: 필수 (API 통신)

**권장 사양**:
- **OS**: Windows 11 (64-bit)
- **RAM**: 8GB 이상
- **디스크 공간**: 1GB
- **프로세서**: Intel Core i5 또는 AMD Ryzen 5 이상
- **인터넷**: 고속 연결 (LLM API 응답 속도 향상)

**필수 소프트웨어**:
- ✅ Microsoft WebView2 Runtime (자동 설치됨)
- ✅ .NET Framework 4.7.2 이상 (Windows 10에 포함)

---

### 1.2 설치 방법

TriFlow AI는 두 가지 설치 파일을 제공합니다:

#### A. MSI 설치 파일 (권장 - 기업 환경)

**특징**:
- Windows Installer 표준 형식
- GPO (Group Policy)를 통한 배포 가능
- 자동 업데이트 지원
- 설치/제거 기록 관리

**설치 단계**:
1. `TriFlow AI_0.1.8_x64_en-US.msi` 파일을 다운로드합니다.
2. 파일을 더블 클릭하여 실행합니다.
3. 설치 마법사의 지시를 따릅니다:
   - **라이선스 동의** → 동의함 체크
   - **설치 경로** → 기본값 (`C:\Program Files\TriFlow AI`)
   - **시작 메뉴 바로가기** → 생성 (권장)
4. "설치" 버튼을 클릭하고 완료될 때까지 기다립니다.
5. 설치 완료 후 "완료" 버튼을 클릭합니다.

**자동 설치 (관리자용)**:
```powershell
# 자동 설치 (무인 모드)
msiexec /i "TriFlow AI_0.1.8_x64_en-US.msi" /qn

# 로그와 함께 설치
msiexec /i "TriFlow AI_0.1.8_x64_en-US.msi" /qn /l*v install.log
```

#### B. NSIS 설치 파일 (개인 사용자용)

**특징**:
- 더 작은 파일 크기 (3.0 MB vs 3.9 MB)
- 빠른 설치 속도
- 휴대용 설치 옵션

**설치 단계**:
1. `TriFlow AI_0.1.8_x64-setup.exe` 파일을 다운로드합니다.
2. 파일을 더블 클릭하여 실행합니다.
3. Windows Defender SmartScreen 경고가 나타나면:
   - "추가 정보" 클릭
   - "실행" 클릭
4. 설치 마법사 진행:
   - **설치 경로** → 기본값 또는 커스텀
   - **바로가기 생성** → 데스크톱/시작 메뉴 (선택)
5. "Install" 버튼을 클릭하고 완료를 기다립니다.

**설치 확인**:
- 시작 메뉴에서 "TriFlow AI Platform" 검색
- 데스크톱 바로가기 확인 (생성한 경우)

### 1.3 초기 설정

#### A. 첫 실행

1. TriFlow AI를 실행하면 **Settings** 페이지가 자동으로 열립니다.
2. **API 키 설정**이 필요합니다 (앱 사용을 위한 필수 단계).

#### B. Anthropic API 키 설정

**API 키 발급** (Anthropic 계정 필요):
1. [https://console.anthropic.com/](https://console.anthropic.com/) 접속
2. 로그인 또는 회원가입
3. "API Keys" 메뉴로 이동
4. "Create Key" 버튼 클릭
5. 키 이름 입력 (예: "TriFlow Desktop")
6. **API 키 복사** (sk-ant-api03-으로 시작)

**TriFlow AI에 API 키 입력**:
1. Settings 페이지에서 "Anthropic API Key" 입력 칸 찾기
2. 복사한 API 키 붙여넣기
3. "Save Settings" 버튼 클릭
4. ✅ 성공 토스트 메시지 확인

**주의사항**:
- ⚠️ API 키는 절대 공유하지 마세요 (비밀번호와 동일하게 취급)
- ⚠️ GitHub 등 공개 저장소에 업로드 금지
- ⚠️ 키가 유출되었다면 즉시 Anthropic Console에서 키를 삭제하세요

#### C. 연결 테스트

API 키 저장 후:
1. **Chat Interface** 페이지로 이동
2. 간단한 메시지 입력: "안녕하세요"
3. AI 응답이 정상적으로 표시되면 설정 완료! 🎉

**응답이 없다면**:
- Settings로 돌아가서 API 키 재확인
- [FAQ.md](FAQ.md#api-키-설정-문제) 참조

## 2. Chat Interface

Chat Interface는 TriFlow AI의 **핵심 기능**으로, 자연어로 AI와 대화하며 데이터 판단, 워크플로우 실행, 인사이트 생성 등을 수행할 수 있습니다.

### 2.1 기본 사용법

#### 화면 구성

```
┌─────────────────────────────────────────────┐
│  TriFlow AI Platform  [Chat] [Workflow] ... │
├─────────────────────────────────────────────┤
│                                             │
│  [사용자] 최근 판단 결과 요약해줘            │
│                                             │
│  [AI] 📊 최근 7일간 판단 결과:              │
│       - 총 판단 횟수: 150회                 │
│       - 성공률: 94.7%                       │
│       - 평균 처리 시간: 1.2초               │
│                                             │
│  [사용자] ...                               │
│                                             │
├─────────────────────────────────────────────┤
│  [메시지 입력...]                  [전송] │
└─────────────────────────────────────────────┘
```

#### 메시지 입력 방법

1. **텍스트 입력**:
   - 하단 입력창에 메시지 입력
   - "전송" 버튼 클릭 또는 `Enter` 키

2. **멀티라인 입력**:
   - `Shift + Enter`로 줄바꿈
   - 긴 규칙이나 복잡한 요청시 유용

3. **단축키**:
   - `Ctrl + L`: 대화 기록 초기화
   - `Ctrl + /`: 명령 팔레트 열기 (향후 추가 예정)

---

### 2.2 AI 판단 요청

#### A. 간단한 판단 요청

**예시 1: 데이터 정상 여부 확인**
```
사용자: 온도가 85도인데 정상인가요?

AI: ⚠️ 경고 수준입니다.
• 온도: 85°C
• 권장 범위: 20-70°C
• 상태: 비정상 (과열)
• 권장 조치: 즉시 냉각 장치 점검 필요
```

**예시 2: 복잡한 조건 판단**
```
사용자: 다음 데이터가 정상인지 확인해줘:
- 온도: 45도
- 습도: 65%
- 진동: 50Hz

AI: ✅ 모든 항목이 정상 범위 내에 있습니다.

상세 분석:
• 온도: 45°C ✅ (권장: 20-70°C)
• 습도: 65% ✅ (권장: 30-80%)
• 진동: 50Hz ✅ (권장: 0-100Hz)

종합 평가: 정상 작동 중
```

#### B. Rule Engine + LLM 하이브리드 판단

TriFlow AI는 **하이브리드 판단 방식**을 사용합니다:

1. **1단계: Rule Engine 실행** (빠르고 정확)
   - 사전 정의된 규칙 평가
   - 신뢰도 ≥ 0.7이면 즉시 반환

2. **2단계: LLM 보완** (복잡한 케이스)
   - Rule Engine 실패 또는 저신뢰도시
   - LLM으로 추가 분석

**결과 표시**:
```
[AI 판단 결과]
• 방법: Rule Engine (신뢰도 95%)
• 결과: 정상
• 처리 시간: 0.3초
• 설명: temperature <= 70 && humidity <= 80
```

---

### 2.3 대화 기록 관리

#### A. 대화 기록 보기

- **자동 저장**: 모든 대화는 로컬 데이터베이스에 자동 저장
- **영구 보관**: 앱을 종료해도 대화 기록 유지
- **페이지 새로고침 후 복원**: 앱 재시작시에도 이전 대화 표시

#### B. 대화 초기화

**방법 1: 키보드 단축키**
- `Ctrl + L` 입력

**방법 2: 채팅 입력 (향후 추가)**
```
사용자: /clear

AI: 대화 기록이 초기화되었습니다.
```

#### C. 대화 내보내기 (향후 추가)

```
사용자: 이 대화 내용 내보내줘

AI: 📄 대화 내용을 Markdown 파일로 저장했습니다:
C:\Users\[사용자]\Downloads\chat_export_20251113.md
```

## 3. Workflow Builder

Workflow Builder는 **n8n 스타일 드래그앤드롭 에디터**로, 복잡한 비즈니스 프로세스를 시각적으로 설계하고 자동화할 수 있습니다.

### 3.1 워크플로우 생성

#### A. 새 워크플로우 만들기

1. 상단 메뉴에서 **"Workflow"** 클릭
2. **"+ New Workflow"** 버튼 클릭
3. 워크플로우 이름 입력:
   - 예: "재고 알림 자동화"
   - 설명 (선택): "재고 5개 미만시 Slack 알림"
4. **"Create"** 버튼 클릭

#### B. 화면 구성

```
┌─────────────────────────────────────────────────────┐
│  Workflow Builder  [재고 알림 자동화]  [저장] [실행]│
├────────────┬────────────────────────────────────────┤
│            │                                        │
│  노드 팔레트│    ┌──────┐                           │
│            │    │START │                           │
│  🔵 Trigger│    └──┬───┘                           │
│  ⚙️ Action │       │                               │
│  🔀 Logic  │    ┌──▼──────┐                        │
│            │    │Judgment │                        │
│            │    └──┬──────┘                        │
│            │       │                               │
│            │    ┌──▼────────┐                      │
│            │    │Send Alert │                      │
│            │    └───────────┘                      │
│            │                                        │
└────────────┴────────────────────────────────────────┘
```

---

### 3.2 노드 추가 및 연결

#### A. 노드 타입

**1. Trigger 노드** (🔵 파란색)
- **시작점**: 워크플로우가 실행되는 조건
- 예시:
  - Schedule Trigger: 매일 오전 9시
  - Webhook Trigger: API 호출시
  - File Watcher: 파일 변경 감지

**2. Judgment 노드** (🟢 녹색)
- **판단 실행**: Rule Engine + LLM
- 입력: 데이터
- 출력: true/false + 설명

**3. Action 노드** (🟡 노란색)
- **작업 수행**: 외부 시스템 연동
- 예시:
  - Send Email
  - Post to Slack
  - Update Database

**4. Logic 노드** (🔀 회색)
- **흐름 제어**: 조건 분기, 반복
- 예시:
  - If/Else
  - Switch
  - Loop

#### B. 노드 추가 방법

**방법 1: 드래그앤드롭**
1. 왼쪽 팔레트에서 원하는 노드 찾기
2. 마우스로 드래그
3. 캔버스에 드롭

**방법 2: 더블 클릭**
1. 캔버스 빈 공간 더블 클릭
2. 노드 검색 메뉴 표시
3. 노드 타입 선택

#### C. 노드 연결

1. **출력 핸들**에서 마우스 드래그 시작 (노드 오른쪽 ● 점)
2. 대상 노드의 **입력 핸들**로 연결 (노드 왼쪽 ● 점)
3. 연결선이 자동으로 생성됨

**연결 규칙**:
- ✅ Trigger → Judgment: 가능
- ✅ Judgment → Action: 가능
- ❌ Action → Trigger: 불가능 (역방향 연결 금지)

---

### 3.3 워크플로우 실행

#### A. 수동 실행

1. 우측 상단 **"실행"** 버튼 클릭
2. 입력 데이터 JSON 입력 (선택):
   ```json
   {
     "temperature": 85,
     "humidity": 60
   }
   ```
3. **"Execute"** 버튼 클릭
4. 실행 결과 확인:
   - 각 노드별 출력 데이터
   - 실행 시간 및 성공 여부

#### B. 스케줄 실행 (향후 추가)

```
사용자: 이 워크플로우를 매일 오전 9시에 실행해줘

AI: ✅ 스케줄이 등록되었습니다.
• 워크플로우: 재고 알림 자동화
• 실행 시각: 매일 09:00 KST
• 다음 실행: 2025-11-14 09:00
```

### 3.4 시뮬레이션 기능

**실제 실행 전 미리 테스트**하여 워크플로우 동작을 검증할 수 있습니다.

#### A. 시뮬레이션 시작

1. Workflow Builder 화면에서 **"시뮬레이션"** 버튼 클릭
2. 시뮬레이션 패널이 오른쪽에 표시됩니다.

#### B. 시뮬레이션 패널 구성

```
┌─────────────────────────────────────┐
│  시뮬레이션 🎭                      │
├─────────────────────────────────────┤
│  현재 단계: 1/3                     │
│  진행률: [████░░░░░░] 33%           │
│                                     │
│  ┌─────────────────────────────┐   │
│  │ 노드: START                 │   │
│  │ 상태: ✅ 완료               │   │
│  │ 출력: {"workflow_id": ...}  │   │
│  └─────────────────────────────┘   │
│                                     │
│  ┌─────────────────────────────┐   │
│  │ 노드: Judgment              │   │
│  │ 상태: ⏳ 실행 중...         │   │
│  └─────────────────────────────┘   │
│                                     │
│  [다음 단계]  [건너뛰기]  [중지]  │
└─────────────────────────────────────┘
```

#### C. 단계별 실행

**자동 모드**:
- **"다음 단계"** 버튼 클릭시마다 한 노드씩 실행
- 각 노드의 입력/출력 데이터 확인 가능

**연속 실행 모드** (향후 추가):
- **"자동 재생"** 버튼으로 전체 워크플로우 자동 실행
- 각 단계마다 2초 대기

#### D. 시뮬레이션 결과 확인

**성공시**:
```
✅ 시뮬레이션 완료!
• 총 단계: 3개
• 성공: 3개
• 실패: 0개
• 총 실행 시간: 2.5초
```

## 5. Settings

Settings 페이지에서는 TriFlow AI의 모든 환경 설정을 관리할 수 있습니다.

### 5.1 API 키 관리

#### A. Anthropic API 키 설정

**위치**: Settings > API Configuration

1. **"Anthropic API Key"** 입력 필드 찾기
2. API 키 입력 (sk-ant-api03-으로 시작)
3. **"Save Settings"** 클릭
4. 성공 메시지 확인: "✅ API key saved successfully!"

#### B. API 키 검증

**자동 검증**:
- Save Settings 클릭시 자동으로 유효성 검사
- 잘못된 키 형식: "❌ Invalid API key format"

**수동 테스트**:
- Chat Interface에서 간단한 메시지 전송
- 응답이 정상적으로 오면 키가 유효함

#### C. API 키 변경

1. Settings로 이동
2. 새 API 키 입력 (기존 키 덮어쓰기)
3. Save Settings 클릭
4. 앱 재시작 불필요 (즉시 적용)

### 5.2 MCP 서버 연결

**MCP (Model Context Protocol)**: 외부 데이터 소스 및 도구와 연동하는 프로토콜

#### A. 지원 MCP 서버

현재 지원:
- PostgreSQL (데이터베이스)
- Filesystem (파일 시스템)
- Memory (세션 컨텍스트)

향후 추가 예정:
- Slack
- GitHub
- Notion

#### B. 연결 상태 확인

**Settings 화면**:
```

├───────────────┼────────┼──────────┤
```

#### A. 백업 생성

**Chat Interface를 통한 백업**:
```
사용자: 데이터베이스 백업해줘

AI: ✅ 백업이 완료되었습니다!
• 파일: judgify_backup_20251113_143522.db.gz
• 크기: 2.1 MB (gzip 압축)
• 위치: C:\Users\[사용자]\AppData\Local\TriFlow AI\backups\
```

#### B. 백업 목록 조회

```
사용자: 백업 파일 목록 보여줘

AI: 📋 백업 파일 목록 (최신순):
1. judgify_backup_20251113_143522.db.gz (2.1 MB)
2. judgify_backup_20251113_093015.db.gz (2.0 MB)
3. judgify_backup_20251112_183042.db.gz (1.9 MB)

총 3개, 총 용량: 6.0 MB
```

#### C. 백업에서 복구

```
사용자: 가장 최근 백업에서 복구해줘

AI: ⚠️ 복구 전 현재 DB를 안전 백업합니다...
✅ 안전 백업 완료: judgify.db.before_restore

🔄 복구 중...
✅ 복구 완료!
• 복구된 백업: judgify_backup_20251113_143522.db.gz
• 복구 시간: 2025-11-13 15:20:45
```

**주의사항**:
- ⚠️ 복구시 **현재 데이터는 덮어씌워집니다**
- ✅ 안전 백업이 자동으로 생성되므로, 복구 실패시 원상복구 가능

#### D. 자동 정리

- **최근 10개 백업만 유지**
- 오래된 백업은 자동 삭제
- 디스크 공간 절약 (gzip 압축 적용)

### 6.2 커스텀 규칙 작성

**Rule Engine 규칙 문법** (Rhai 스크립트 기반):

#### A. 간단한 규칙

```rhai
// 온도 범위 체크
if temperature >= 20 && temperature <= 70 {
    true  // 정상
} else {
    false  // 비정상
}
```

#### B. 복잡한 조건

```rhai
// AND, OR, NOT 연산자
if (temperature < 50 || temperature > 90) && humidity > 80 {
    false  // 위험
} else if temperature >= 50 && temperature <= 90 {
    true  // 정상
} else {
    false  // 기타
}
```

#### C. 배열/객체 처리

```rhai
// 배열 길이 확인
if sensors.len() > 5 {
    true
} else {
    false
}

// 배열 인덱싱
if sensors[0] > 100 {
    false
} else {
    true
}

// 객체 속성 접근
if data.temperature > 70 && data.humidity > 80 {
    false
} else {
    true
}
```

#### D. 규칙 등록 (향후 추가)

```
사용자: 다음 규칙을 "온도 체크"라는 이름으로 저장해줘:
temperature >= 20 && temperature <= 70

AI: ✅ 규칙이 저장되었습니다!
• 이름: 온도 체크
• 표현식: temperature >= 20 && temperature <= 70
• UUID: a1b2c3d4-e5f6-...
```

### 6.3 성능 최적화

#### A. 캐시 설정

**Memory-First Hybrid Cache** 아키텍처:
- **L1 Cache (메모리)**: 0.001ms GET, 90% 적중률
- **L2 Cache (SQLite)**: 10ms GET, 100% 지속성

**자동 최적화**:
- 자주 사용되는 판단 결과는 메모리에 캐싱
- 10분간 접근 없으면 자동 만료

#### B. 데이터베이스 최적화

**자동 Vacuum** (향후 추가):
```
사용자: 데이터베이스 최적화해줘

AI: 🔧 최적화 중...
✅ 완료!
• 이전 크기: 5.2 MB
• 최적화 후: 3.8 MB (27% 절감)
• 처리 시간: 3.2초
```

#### C. 로그 파일 정리 (향후 추가)

```
사용자: 30일 이상 된 로그 삭제해줘
```

## 7. 문제 해결

### 7.1 일반적인 오류

#### A. "API key not configured" 오류

**증상**: Chat Interface에서 메시지 전송시 에러 발생

**해결 방법**:
1. Settings 페이지로 이동
2. Anthropic API Key 입력
3. Save Settings 클릭
4. Chat Interface에서 재시도

---

#### B. "WebSocket connection failed" 경고

**증상**: Dashboard 데이터가 업데이트되지 않음

**해결 방법**:
1. 인터넷 연결 확인
2. 방화벽에서 TriFlow AI 허용 확인
3. 앱 재시작

---

#### C. "Workflow execution timeout" 오류

**증상**: 워크플로우 실행이 중간에 멈춤

**원인**:
- 외부 API 응답 지연
- LLM 요청 시간 초과
- 네트워크 불안정

**해결 방법**:
1. 워크플로우 재실행
2. 타임아웃 설정 증가 (Settings > Advanced)
3. 네트워크 연결 확인

---

### 7.2 로그 확인

#### A. 로그 파일 위치

**Windows**:
```
C:\Users\[사용자]\AppData\Local\TriFlow AI\logs\
```

**로그 파일 종류**:
- `app.log`: 일반 애플리케이션 로그
- `error.log`: 에러 로그만 필터링

#### B. 로그 레벨

- `DEBUG`: 상세한 디버깅 정보
- `INFO`: 일반 정보 메시지
- `WARN`: 경고 메시지
- `ERROR`: 에러 메시지
