## ❓ 예상 질문 및 답변

### Q1: "Rule Engine과 LLM의 차이가 뭔가요?"

**답변**:
> "Rule Engine은 명확한 수식을 빠르게 평가합니다.
> 예를 들어 'temperature > 90'같은 조건은 밀리초 단위로 판단됩니다.
>
> LLM은 모호하거나 복잡한 상황을 자연어로 판단합니다.
> 예를 들어 '이 제품이 프리미엄 품질인가?'같은 질문이죠.
>
> Judgify는 두 가지를 순차적으로 실행해서 최적의 결과를 얻습니다."

### Q2: "실제 프로덕션에서 사용 가능한가요?"

**답변**:
> "현재는 Desktop App 프로토타입 단계입니다.
>
> Ver2.0 Final 계획에서는 9개 마이크로서비스로 확장하여
> - Kubernetes 배포
> - PostgreSQL + pgvector로 RAG 구현
> - 자동학습 시스템 (Learning Service)
> - 프로덕션급 모니터링
>
> 등을 추가할 예정입니다."

### Q3: "어떤 산업에 적용할 수 있나요?"

**답변**:
> "제조업 품질 관리, 금융 리스크 평가, 콘텐츠 모더레이션,
> 의료 진단 보조, HR 채용 심사 등
>
> 복잡한 판단 기준이 있는 모든 분야에 적용 가능합니다."

### Q4: "API 호출 비용은 어떻게 되나요?"

**답변**:
> "Rule Engine은 무료입니다. (로컬 실행)
>
> LLM은 OpenAI API 호출시 비용이 발생하지만,
> Rule이 먼저 실행되고 필요할 때만 LLM을 호출하므로
> 순수 LLM 방식 대비 50-70% 비용 절감이 가능합니다."

### Q5: "자동학습 기능은 어떻게 작동하나요?"

**답변**:
> "현재 프로토타입에는 미구현이고, Ver2.0에서 추가됩니다.
>
> Learning Service가 사용자 피드백(👍👎)을 수집하고,
> - Few-shot 학습 (유사 예시 10-20개)
> - 자동 Rule 추출 (3개 알고리즘)
> - pgvector 기반 유사 사례 검색
>
> 으로 점점 정확도가 향상됩니다."

## 🛑 긴급 상황 대응

### 문제 1: 앱이 로드되지 않음
**해결**:
```bash
# 터미널 재시작
Ctrl+C
npm run tauri:dev
```

### 문제 2: 데이터베이스 비어있음
**해결**:
1. Settings → MCP 서버 → SQLite 연결 확인
2. 또는 터미널에서:
```bash
# DB 파일 삭제 후 재시드
rm %APPDATA%\Judgify\judgify.db
# 앱 재시작하면 자동 시드
```

### 문제 3: OpenAI API 에러
**해결**:
- Settings 페이지에서 API 키 재입력
- "테스트" 버튼으로 연결 확인
- 또는 데모 중에는 "API 연결 문제로 실제 응답은 생략하겠습니다" 멘트

### 문제 4: 차트 데이터 없음
**해결**:
- Workflow Builder에서 워크플로우 1-2개 실행
- Dashboard로 돌아가서 새로고침 (F5)
